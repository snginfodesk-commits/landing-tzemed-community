# דף נחיתה — קהילת צמד ברזל

דף נחיתה לאיסוף לידים לקהילת הוואטסאפ של צמד ברזל, מיועד לתנועה ממומנת מפייסבוק ואינסטגרם.

## מבנה

| קובץ | תפקיד |
|---|---|
| `index.html` | דף הנחיתה. HTML/CSS/JS בקובץ אחד, בלי תלויות חיצוניות |
| `thank-you.html` | דף התודה. CTA יחיד לוואטסאפ + קרוסלת הודעות מהקהילה |
| `assets/fonts/` | Heebo מאוחסן מקומית (woff2) |
| `assets/*.webp` | תמונות מיטביות שמוגשות בדף |
| `assets/*.jpg`, `logo-gold.png` | מקורות ברזולוציה מלאה — לא מוגשים, נשמרים לגיבוי |

## זרימת הליד

`index.html` → POST ל‑webhook של n8n → הפניה ל‑`thank-you.html?name=…` → כפתור הצטרפות לוואטסאפ

השליחה עוטפת את ה‑POST ב‑`.finally()`, כך שכשל של ה‑CRM **לעולם לא חוסם** את הגולש מלהגיע לדף התודה. ליד שנכשל נשמר ב‑`localStorage` תחת `tb_leads_pending`.

הליד מתויג ב‑CRM עם `source: "קמפיין ממומן"`.

## החלטות טכניות שכדאי להכיר לפני עריכה

רוב התנועה מגיעה מ‑**webview של אינסטגרם/פייסבוק**, שמתנהג אחרת מדפדפן רגיל:

- **בלי `IntersectionObserver`** — ה‑callbacks שלו לא נורים באופן אמין ב‑webview. סרגל ה‑CTA הדביק משתמש ב‑`scroll` listener רגיל.
- **בלי `requestAnimationFrame`** לאלמנטים קריטיים — הוא עלול להיחנק. הקרוסלה בנויה על גלילה טבעית עם `scroll-snap`, כך שהחלקה באצבע עובדת גם בלי JS.
- **בלי AVIF** — התמיכה ב‑webview לא אחידה. WebP נתמך בכל מקום רלוונטי.

### פונטים

Heebo הוא **פונט משתנה (variable font)** — קובץ אחד לכל תת‑קבוצה מכסה את כל המשקלים 100–900. לכן יש רק שני קבצים (`hebrew`, `latin`) ולא אחד לכל משקל.

הפונטים מאוחסנים מקומית בכוונה: ה‑stylesheet של Google Fonts היה המשאב היחיד בדף שחסם רינדור, והוא עלה שתי התקשרויות DNS+TLS לצד שלישי.

## עריכה

הכל סטטי. פותחים את `index.html` בדפדפן — אין שלב build, אין תלויות, אין `npm install`.

לתצוגה מקומית עם נתיבים תקינים:

```bash
python3 -m http.server 8931
# http://localhost:8931
```

## העלאה לאוויר

הדף סטטי לחלוטין. בכל שירות אירוח, מומלץ לוודא:

- **Brotli או Gzip** — מקטין את ה‑HTML מ‑28KB ל‑כ‑8KB
- **`Cache-Control` ארוך על `assets/`** — התמונות והפונטים לא משתנים
